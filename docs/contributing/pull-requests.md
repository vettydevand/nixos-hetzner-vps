# Pull Requests

We welcome contributions to this project. To make the process as smooth as possible, please follow these guidelines when submitting a pull request.

## Before You Start

- **Search for existing issues and pull requests.** It's possible that someone else has already reported the issue or is working on a similar feature.
- **Open an issue to discuss your idea.** This is especially important for large changes. It allows us to discuss the proposed changes and ensure that they are in line with the goals of the project.

## Creating a Pull Request

1. **Fork the repository.**
2. **Create a new branch for your changes.**
3. **Make your changes.**
4. **Format and lint your code.** See the [Development guide](development.md) for instructions.
5. **Commit your changes.** Please write a clear and concise commit message.
6. **Push your changes to your fork.**
7. **Open a pull request.**

## What to Expect

After you open a pull request, we will review your changes and provide feedback. We may ask you to make changes before we can merge your pull request.

Once your pull request is approved, we will merge it into the `main` branch. Thank you for your contribution!
